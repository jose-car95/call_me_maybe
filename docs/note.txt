
========= Poceso de generacion del LLM: =========
Prompt -> Tokenization -> Input IDs -> LLM -> Logits -> Next Token Selection


Logits: El modelo produce una distribucion de probabilidad(logits), sobre
    todos los posibles tokens siguientes

========= CONCEPTOS A DOMINAR ===========

1. LLM
    Un LLM es un modelo que predice el siguiente token. No “piensa” como una persona ni ejecuta código.
    Recibe texto convertido en números y devuelve puntuaciones sobre cuál podría ser el siguiente token.

2. Tokenización
    El texto se divide en tokens. Un token puede ser una palabra, parte de una palabra, un espacio o un símbolo.

    Ejemplo simplificado:

    "What is the sum of 2 and 3?"
    puede convertirse en algo parecido a:
    ["What", " is", " the", " sum", " of", " 2", " and", " 3", "?"]
    Luego cada token se transforma en un número: los input_ids.

3. Logits
    El modelo no devuelve directamente texto.
    Devuelve una lista de puntuaciones, una por cada token posible del vocabulario.
    Esas puntuaciones se llaman logits.

    En el SDK del proyecto, esto se hace con llm_sdk/llm_sdk/__init__.py:90:
    get_logits_from_input_ids(input_ids)
    Ese método nos dice: “después de estos tokens, qué tokens parecen más probables”.

4. Decodificación
    Decodificar es elegir el siguiente token usando los logits.
    Normalmente se elige el token con mayor puntuación o se hace sampling.

    Proceso mental:
    prompt -> tokens -> ids -> modelo -> logits -> elegir siguiente token

    Y se repite muchas veces.

5. Function Calling
    Function calling significa transformar lenguaje natural en una estructura ejecutable por un programa.
